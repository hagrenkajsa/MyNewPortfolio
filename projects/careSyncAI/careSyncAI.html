CareSyncAI

A mobile application wth the purpose to leverage generative AI to enhence documentation and communication within the elderly care.

Project Overview

Project Background

Lack of communication and documentation in elderly care.

Elderly care faces major challenges with documentation and communication due to limited time and a lack of resources. This often leads to miscommunication and misunderstandings, as there are no effective systems in place to manage these issues.

Task

Leveraging new technology to address the problem area.

In a team of five, we were given the opportunity to choose a problem area of our own interest within the area of health care and develop a solution using new technology techniques. The project was carried out in collaboration with the consultancy firm Daresay.

Project summary

An agile UX project focused on improving communication and documentation in elderly care through a mobile app powered by generative AI.

The project followed an agile process, where we first identified a lack of communication and documentation within elderly care, based partly on our own experiences and stories from friends and family. To better understand the problem, we interviewed people working in the field and defined the core challenges.

We then developed the idea of creating a mobile application that enables documentation, planning, and communication with the help of generative AI. In this project, we integrated OpenAI’s language model to support those features. Another useful functionality we implemented was text-to-speech and speech-to-text, making it easier for caregivers to document and access information when their hands are occupied.

The project resulted in a well-developed high-fidelity prototype as well as a React Native mobile application, where we successfully demonstrated the core features and integrations we had planned.

Technical Implementation

What We Implemented

A native mobile application was developed to demonstrate our AI features

To complement our prototype, we built a functional mobile application that showcased our key features powered by OpenAI’s API. The main functionalities were fully implemented and worked in real time with real interactions.

The features included one where the user can select a documentation category and get a summary of all related documents. The other two features were text-to-speech and speech-to-text with the purpose to simplify writing and reading patient documentation, schedule and notifications.

Setup and Structure

For project planning and code management, we used Azure DevOps.

The project was set up and managed using the cloud-based service Azure DevOps to facilitate planning, collaboration, code management, and software deployment. We used Azure Repos for version control, providing a Git repository for storing and managing the source code, and Azure Boards together with Azure Sprints for project planning and task management using a Kanban approach.

Frontend

TypeScript and React Native

For the development of the mobile application, we used the JavaScript framework React Native together with the Expo development platform. TypeScript was used to add type safety to the project, which helped facilitate communication between the frontend application and the server — both written in TypeScript.

Backend

Node.js to communicate with OpenAIs API

For the project’s backend development, we chose Node.js to handle communication with OpenAI’s API. The choice of Node.js was based on its powerful server environment built on JavaScript. By using Node.js, we were able to execute TypeScript code on the server side.

Our Node.js server includes functions that communicate with OpenAI’s API, enabling the generation of documentation summaries, transcription of recorded speech, and text-to-speech conversion. For this purpose, three endpoints were implemented: ‘generate-summary’, ‘tts’ (text-to-speech), and ‘stt’ (speech-to-text).

Data base

Firebase Firestore

For this project, we used Firebase Firestore, a cloud-based NoSQL database that stores data in a document format. The database mainly stores information about the users (patients). There is a collection that contains all users, and each user document includes several fields such as name, address, personal ID number, and care decisions.

This information is used in the frontend to display details about a user that a home care worker is visiting. Another important part of the database is the “notes” collection under each user. This collection stores all documentation created for that specific user, including the category, content, and date. These notes are later used for generating summaries.

Generative AI

We used OpenAI’s API to implement the AI summarization, text-to-speech and speech-to-text features in our application.

As mentioned, the main functionality of our application is a feature that allows the user to get a summary of documentation within different categories of interest. To make this possible, we used OpenAI’s language model. The endpoint for creating summaries takes the patient’s ID and the chosen category, then generates a summary from all the documentation within that specific category.

For the text-to-speech (TTS) implementation, OpenAI’s “Echo” API was used. The user’s text is sent to the Node.js server and then forwarded to the API, which returns the audio. The server converts the audio to base64 format, sends it back to the client, and plays it as a sound file.

For speech-to-text (STT), we used Expo-AV to record the user’s voice and convert it to base64. The encoded audio file is then sent to the server, where OpenAI’s “Whisper” API transcribes the audio and converts it into text.
